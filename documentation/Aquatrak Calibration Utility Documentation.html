Aquatrak Sensor Head Calibration Software

Developer: Winston Hensley

Release Date: 04/25/2015

IDE: LabVIEW 2014 (32-bit)

 

 

Refer to requirements & design documentation for details of this systems development lifecycle.

 

The Aquatrak Sensor Head Calibration Software is based on the Continuous Measurement and Logging (NI-DAQmx) template and acquires measurements continuously while logging them to disk. After the calibration is complete, the software allows the operator to print a label and save the summary and meta data to disk in an ASCII and XML format. The program executes five loops in parallel:

 

         Event handling (Main.vi)The Event Handling Loop that produces messages (String) based on front panel events, such as the user clicking Start or Settings.

         User interface messaging (Main.vi)A Message Handling Loop that receives messages from the Event Handling Loop and responds by sending messages to the other Message Handling Loops.

         Acquiring data (Acquisition.lvlib:Acquisition Message Loop.vi)A Message Handling Loop that continuously acquires data.

         Logging data (Logging.lvlib:Logging Message Loop.vi)A Message Handling Loop that continuously logs acquired data.

         Displaying data (Main.vi)A While Loop that updates the waveform chart with acquired data.

 

 

 

 

Calibrations are only to be performed at the NOAA facility in Chesapeake, VA using the National Instruments (NI) hardware which has been designed for this use. An 8 channel NI 9188 CompactDAQ is utilized to communicate with x2 NI 9205 Analog Input modules and x3 NI 9265 Analog Output modules. The Ethernet port on this device allows the Aquatrak Sensor Head Calibration Software to communicate with this data acquisition system and the DB-9 Serial Port to communicate with the Aquatrak controllers output. This project features a Settings dialog box (Settings.lvlib) you can use to configure the application. Default Settings found below. Max AI Sample Rate(Hz) when using 12 thermistors is 19200S/s. There is currently no way or requirement to use less than 12 thermistors to measure the calibration boxs thermal gradient. Maximum and minimum voltages on the ADC are +-5V based on temperature resolution requirements.

 

Excitation Current and the number of input samples are kept sufficiently low to reduce any self-heating that could occur during the excitation of the thermistors in the system.

 

 

 

This project is based on the Queued Message Handler template. Refer to the Queued Message Handler template and its documentation, available from the Create Project dialog box, for information about how this template works.

 

The NI 9205 ADC module is limited to +-5VDC to adhere to voltage measurement requirements. The calibration line of the Aquatrak sensor is used to trigger the acquisition of temperature, ring time and level measurements but is a full +12VDC signal. It was found necessary to use a voltage divider to reduce the voltage to ~2.5VDC which still does an excellent job of picking up the transducers pulse for triggering purposes.
